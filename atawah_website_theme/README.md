# Atawah Website Theme — لأودو 19

موديول أودو يعيد بناء الصفحة الرئيسية لموقعك على شكل تصميم **ATAWAH**
(Financial Consultancy & Auditing)، مبني بالكامل بأدوات أودو القياسية
(Website Builder / QWeb / SCSS / OWL publicWidget) بحيث يبقى قابلاً
للتعديل من محرر الموقع مباشرة، وليس صفحة HTML جامدة.

## التثبيت

1. انسخ مجلد `atawah_website_theme` إلى مجلد الإضافات (`addons-path`) الخاص بسيرفر أودو 19.
2. فعّل **وضع المطوّر** ثم اذهب إلى: **التطبيقات ▸ تحديث قائمة التطبيقات**.
3. ابحث عن `Atawah Website Theme` وقم بتثبيته.
4. افتح الموقع الإلكتروني من الواجهة الأمامية — ستجد الصفحة الرئيسية الجديدة مباشرة.

> **ملاحظة عن قاعدة البيانات المتعددة المواقع (Multi-Website):** الموديول
> يربط الصفحة الرئيسية والقوائم بالموقع الافتراضي (`website.default_website`).
> إذا كان لديك أكثر من موقع، عدّل `data/website_page_data.xml` و
> `data/website_menu_data.xml` لتأشير الموقع الصحيح.

## أين تُعدّل كل جزء من التصميم؟

| ما تريد تغييره | أين تجده |
|---|---|
| نصوص وعناوين أي قسم (Hero, About, Why...) | مباشرة من محرر الموقع (اضغط على النص وعدّله) |
| الخدمات الست وترتيبها | **الموقع الإلكتروني ▸ Atawah ▸ Services** من الباك اند |
| خطوات العمل الأربع | **الموقع الإلكتروني ▸ Atawah ▸ Process Steps** من الباك اند |
| الألوان (الكحلي/الذهبي) | `static/src/scss/atawah_variables.scss` |
| تفاصيل الأنماط والتخطيط | `static/src/scss/atawah_theme.scss` |
| روابط القائمة العلوية | **الموقع الإلكتروني ▸ التخصيص ▸ القوائم** أو `data/website_menu_data.xml` |
| الهيدر / الفوتر | `views/website_page_templates.xml` |
| ترتيب أقسام الصفحة الرئيسية | `data/website_page_data.xml` (عدّل ترتيب أسطر `t-call`) |
| بيانات فورم التواصل (البريد/الهاتف) | `views/snippets/s_atawah_contact.xml` |

## ملاحظة مهمة عن الألوان والخطوط

الأكواد اللونية المستخدمة (كحلي داكن + ذهبي) **تقديرية**، لأنني لم أتمكن من
الوصول لملف الـ CSS المُجمَّع الفعلي لموقع atawah.vercel.app (الأدوات
المتاحة لدي تقرأ محتوى الصفحة كنص فقط، دون تحميل الأصول الثنائية أو
تنفيذ الجافاسكريبت الخاص به). إن كان لديك:
- ملف هوية بصرية (Brand Guidelines)، أو
- الشعار بصيغة SVG/AI، أو
- إمكانية مشاركة الأكواد اللونية (Hex) مباشرة من مصمم الموقع

أرسلها وسأضبط `atawah_variables.scss` لتطابق الهوية الفعلية 100% بدلاً من التقدير.

## البنية التقنية باختصار

- **9 أقسام (Snippets)** مستقلة في `views/snippets/`، كل قسم مسجّل أيضًا في
  لوحة الـ Building Blocks (`views/snippets_registry.xml`) ليمكن سحبه على
  أي صفحة أخرى غير الرئيسية.
- **موديلان** (`atawah.service` و `atawah.process.step`) لإدارة الخدمات
  وخطوات العمل من الباك اند بدل تعديل XML يدويًا في كل مرة.
- **كونترولر** بسيط (`controllers/main.py`) يغذّي قسم Insights بأحدث
  تدوينات المدونة تلقائيًا إن كان موديول `website_blog` مثبتًا، وإلا يبقي
  البطاقات الافتراضية كما هي.
- **JS واحد** (`static/src/js/atawah_theme.js`) مبني على `publicWidget`
  القياسي في أودو: هيدر ثابت + قائمة موبايل + عدادات متحركة + ظهور تدريجي
  للبطاقات عند التمرير + تحميل Insights.
- فورم التواصل يستخدم آلية `/website/form/` المدمجة في أودو (بدون أي كود
  Python إضافي)، ويُنشئ Lead في `crm.lead` مباشرة (لن يعمل إلا إذا كان
  تطبيق CRM مثبتًا؛ إن لم يكن مثبتًا غيّر `data-model_name` إلى `mail.mail`
  في `s_atawah_contact.xml`).
